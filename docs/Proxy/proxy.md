## 프록시
> 본 문서에서는 프록시 과정을 설명합니다.

1. *.*.optics.run과 *.optics.run으로 들어오는 모든 서비스 대상의 요청을 수신하는 서버를 띄웁니다. 앞으로 이 서버는 클라이언트의 모든 HTTP 소켓 요청을 수신합니다.
2. 요청을 수신하면 Host 헤더를 파싱해 다음과 같은 형태로 분리합니다.
- Host: service.workspace.optics.run
3. 도메인에 포트가 포함된 요청이라면, 포트 지정을 무시합니다.
4. 프록시 서버에서 소켓의 고유 토큰을 생성하고 Hub로 이번 요청의 연관된 정보를 전송합니다.의
- 전송 내용은 host 헤더에서 파싱한 서비스 서브도메인과 워크스페이스 서브도메인, 프록시 서버에서 생성한 토큰입니다.
